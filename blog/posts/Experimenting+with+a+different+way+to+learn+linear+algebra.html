Experimenting with a different way to learn linear algebra

I should begin with a disclaimer: this is an experiment, but, last week, I started working on a new library, linalgebra, designed as a teaching aide. The algorithms are, almost certainly, uniformly non-optimal. That's not the point, though; I'm curious as to whether taking the time to implement concepts in code will help me learn.

Linear algebra is uniquely suited to this task in ways that other fields, like Calculus, aren't. For one, symbolic math is a pain, even with existing libraries, while linear algebra can be purely numeric and built upon standard data types. Meanwhile, it is imperative that any engineer has a firm grasp on at least basic concepts.

As for the library, it's all written in CoffeeScript, my current favorite language, although, in retrospect (and if I didn't have so many other projects), a project like this one could have been a great opportunity to also learn a new language.

So far, it seems to be helping at least a little, although I now feel like everything math-related I'm analyzing through the prism of Computer Science (for the record, that's not necessarily a bad thing). Still, the jury remains out on whether this project will make any difference for me, and I won't really know that until the day of the final.

At any rate, the code is on GitHub. If it looks like I've abandoned it, please bug me about it on Twitter.
